## Summary

f_vortex is a single fixed-point vector field generator. One `jit.gl.pix @type float32` codebox computes a continuous convergence/curl field, encodes it as RG (0.5 = zero vector), and outputs an f_vecfield texture. Four optional modulation inlets (cx, cy, convergence, curl) each offset their corresponding param additively. The patcher follows standard Vsynth generator conventions — no dual-mode, no source texture.

Core field math is validated in `vortex-scratch.maxpat`. The scratch patch used `1/(1+r*f)` falloff — the real codebox uses `exp(-r*f)` which is cleaner (always positive, no ring singularity).

## Architecture Decisions

### ADR 1: exp(-r * falloff) for field decay

**Context:** Scratch patch used `1/(1+r*f)`. At negative falloff values this created a ring singularity — denominator crosses zero, field flips sign, visible circular artifact.

**Decision:** Use `exp(-r * falloff)`. Clamp falloff to 0.0 minimum in codebox. Exponential is always positive, smooth, well-behaved at all param values. Falloff = 0 → uniform field (no decay). High falloff → tight concentration near fixed point.

**Alternatives considered:**
- `1/(1+r*f)` with clamped positive falloff: avoids singularity but is harder to calibrate — asymptotic at large r, doesn't reach zero cleanly.
- Power law `(1 - r)^f`: only defined for r ≤ 1.0, clipping behavior at edges.

**Consequences:**
- Falloff param range: 0.0–10.0. Default 2.0.
- Scratch patch behavior at negative falloff was an artifact — not a feature. Exp falloff will look different; calibrate defaults fresh.

---

### ADR 2: Additive modulation with per-inlet _amt scaling

**Context:** Four params are modulation targets (cx, cy, convergence, curl). Each has a dedicated texture inlet.

**Decision:** Additive modulation — `param_effective = param_dial + sample(inN).x * amt`. Each inlet has a corresponding `_amt` dial. vs_black (0.5 decoded = 0.0) produces zero offset regardless of `_amt`. `_amt = 0` default means modulation inlets are inert until explicitly engaged.

**Rationale:** Consistent with f_ library convention (f_lens field_amt/surface_amt pattern). Additive allows the dial to set a base value and the modulation to ride on top — performable without resetting the base.

**Consequences:**
- 4 extra dials (_amt params). Total: 9 params + bypass.
- Modulation sampling uses `.x` channel (luma/red) — conventional for scalar modulation in f_ family.
- Decoding: `(sample(inN, norm).x - 0.5) * 2.0 * amt` gives signed offset in [-amt, +amt].

---

### ADR 3: Generator only — no source/processor duality

**Context:** f_vortex produces a field texture. It has no use for an incoming image.

**Decision:** Inlet 0 receives bang and control messages only. No texture passthrough, no vs_inState, no bypass that outputs source. Bypass outputs neutral field (all 0.5).

**Consequences:**
- No `routepass jit_gl_texture` needed on inlet 0 — only `jit_matrix` for control.
- Patcher is simpler than dual-mode patches (no vs_inState, no src_mode).
- Output type is always float32 regardless of upstream.

---

### ADR 4: Build script generates patcher JSON

**Decision:** Python build script at `~/Vsynth/patterns/build_f_vortex.py`. Consistent with f_lens, f_grain, f_masonry pattern.

**Consequences:**
- Patcher never hand-edited after build script exists.
- Build script written after codebox confirmed working.
- Scratch patch (`vortex-scratch.maxpat`) remains as reference — not deleted.
